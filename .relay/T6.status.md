## 偏离/疑问
- **key set/check 作用域为 keychain**:依 main.rs Key 命令 help「管理 keychain 中的密钥」与 S4「写入 keychain」。非 keychain 的 key_ref(bw/env)由 `env`/`code` 实际取用时校验,不在 `key check` 覆盖。若希望 `key check` 跟随 provider 的 key_ref 后端,请指出。
- **key set 隐藏输入需真实 tty**:rpassword 打开 `/dev/tty`,非交互(管道)环境报 "Device not configured"——这是防误喂的安全设计,真机交互正常;其 keychain 写入路径已由 T4 `keychain_roundtrip` 真机覆盖。
- `--profile` 覆盖 key_ref 自带 profile(CLI 赢),与三层合并「用户输入优先」一致。
- refresh 缓存的 138 条只各含 display_name(PROPOSAL-001 A);load_merged 仍只 enrich 已知 4 家。
